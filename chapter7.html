Chapter 07

Understanding Software

From operating systems to enterprise software — why it all matters to managers

📖
Vocabulary
Software
A computer program, or collection of programs — a precise set of instructions that tells hardware what to do. Without software, hardware is just expensive metal and plastic. Software is what enables a device to browse the web, process payroll, recognize your voice, or run a factory. It is everywhere: a car has 10+ million lines of code; an EV has 100+ million lines — more than Windows or macOS.
🧠 Analogy: Hardware is the piano. Software is the sheet music. Without the music, the piano just sits there.
Source: Gallaugher, Ch. 7
Hardware
The physical components of information technology — equipment you can physically touch. This includes computers, storage devices, networking equipment, input devices (keyboard, mouse), and output devices (monitor, printer). Hardware executes software instructions but cannot do anything useful on its own without software directing it.
Source: Gallaugher, Ch. 7
Operating System (OS)
Software that controls a computer's hardware and provides a common set of standards for developing and running applications. The OS manages memory, files, input/output devices, and security. Examples: Windows, macOS, iOS, Android, Linux. Every computing device — phones, servers, game consoles, smart appliances — runs some form of OS.
🧠 Analogy: The OS is the building manager of an office. It controls the electricity, elevators, and hallways (hardware), so that tenants (applications) can just focus on their own work without worrying about the building's infrastructure.
Source: Gallaugher, Ch. 7; Microsoft Docs
Graphical User Interface (GUI)
The visual, interactive layer through which users interact with a computing device. GUI elements include windows, scroll bars, buttons, menus, icons, and dialog boxes. The OS enforces a consistent look and feel across all applications — which is why scroll bars and save dialogs look similar in every Windows app. This consistency lowers training costs and reduces user errors. Pronounced "gooey."
🧠 Analogy: The GUI is the cockpit dashboard. Without it, you'd have to type raw commands to fly the plane. With it, you just press labeled buttons.
Source: Gallaugher, Ch. 7
Firmware
Software stored permanently on non-volatile memory chips (ROM) inside a device, rather than on a hard drive. Firmware controls low-level hardware functions and often holds the entire OS for small or specialized devices (like routers, printers, or smart appliances). The BIOS — the base-level code that boots a computer — is a classic example of firmware.
🧠 Analogy: Firmware is the factory-installed "muscle memory" of a device. It knows how to turn on and find the OS before any user touches it.
Source: Gallaugher, Ch. 7
Embedded Systems
Special-purpose software (often stored as firmware) designed and built into physical products to make them "smarter." Embedded systems enable devices to share usage data, diagnose problems, receive remote updates, or communicate with other systems — without a user needing to intervene. Found in everything from elevators and cars to pacemakers and smart thermostats.
🧠 Analogy: An embedded system is like a built-in doctor inside a machine — monitoring its own vitals and calling for help before anything actually breaks.
Source: Gallaugher, Ch. 7 (Otis Elevator example)
Desktop Software
Applications installed on a personal computer, typically designed for a single user performing a specific task. Examples: Microsoft Word, Excel, Photoshop, Chrome, video games. Desktop software runs "on top of" the operating system and uses the OS's services (like saving files, displaying graphics) to do its job.
Source: Gallaugher, Ch. 7
Enterprise Software
Applications that address the needs of multiple simultaneous users throughout an organization or workgroup. Unlike desktop software (built for one person), enterprise software manages shared business processes — inventory, payroll, customer records, supply chains. These systems are often costly, complex, and critical to a company's operations.
🧠 Analogy: Desktop software is a single employee's toolbox. Enterprise software is the entire company's shared operating infrastructure — like the plumbing and electrical systems of a building that everyone depends on.
Source: Gallaugher, Ch. 7
ERP — Enterprise Resource Planning
A software package that integrates many core business functions — accounting, finance, inventory, human resources, manufacturing, sales — into one unified system with a shared database. ERP eliminates data silos by ensuring all departments see the same, real-time information. Leading vendors: SAP and Oracle. Average large company spend: ~$15 million, with some installations exceeding hundreds of millions.
🧠 Analogy: ERP is the central nervous system of a company. Every department (brain region) sends and receives information through one shared network, so the whole body reacts in coordination.
Source: Gallaugher, Ch. 7; SAP.com
DBMS — Database Management System
Software for creating, storing, organizing, retrieving, and maintaining data. The DBMS sits between the OS and the enterprise applications — it's the shared data layer that multiple programs (ERP, CRM, BI) all read from and write to. When all systems use the same DBMS, a sale recorded by the point-of-sale system automatically updates inventory, accounting, and reporting — with no manual re-entry.
🧠 Analogy: The DBMS is the shared filing room in the middle of an office. Every department drops off and retrieves records from the same place, so everyone's always working from the same files — no duplicates, no conflicts.
Source: Gallaugher, Ch. 7
CRM — Customer Relationship Management
Enterprise software used to manage and analyze customer interactions across sales, marketing, and customer service. A CRM tracks who your customers are, what they've bought, what they've complained about, and what they're likely to buy next. The goal: build and maintain profitable customer relationships. Salesforce is the most well-known CRM vendor.
Source: Gallaugher, Ch. 7; Salesforce.com/crm
SCM — Supply Chain Management
Enterprise software that helps a firm manage its value chain — from raw materials flowing into the firm, through production, to finished products delivered to customers. SCM systems coordinate suppliers, manufacturers, warehouses, and distributors to ensure the right products arrive at the right place at the right time at minimum cost.
🧠 Analogy: SCM is the GPS + logistics coordinator for everything a company makes and moves. It tracks every ingredient from farm to fork.
Source: Gallaugher, Ch. 7
BI — Business Intelligence
Systems that use data collected by other enterprise systems (ERP, CRM, SCM) to generate reports, dashboards, and analyses that support organizational decision-making. BI doesn't create or change data — it reads data from across the organization and turns it into actionable insight. Example: a BI dashboard showing which product lines are most profitable by region.
🧠 Analogy: If ERP is the company's nervous system collecting data, BI is the brain interpreting that data and deciding what to do next.
Source: Gallaugher, Ch. 7
API — Application Programming Interface
Published rules and protocols that allow one software program to request services from another — even across different organizations. APIs let companies turn their products into platforms. Example: Expedia publishes APIs so that partner airlines (United, Aer Lingus) can offer hotel bookings inside their own apps, powered by Expedia's inventory — generating ~$2 billion in additional annual revenue for Expedia.
🧠 Analogy: An API is like a restaurant's order window. You don't need to know how the kitchen works — you just follow the menu (the API rules) to request what you want and receive a result.
Source: Gallaugher, Ch. 7

Key Concepts
Why Managers Must Understand Software
Software is not just an IT department problem — it directly shapes a firm's strategy, cost structure, and competitive position. Managers who understand software can:

• Evaluate the strategic value of technology investments rather than blindly trusting IT vendors
• Anticipate security vulnerabilities created by poorly integrated systems
• Understand the risks of enterprise system failures (FoxMeyer Drug went bankrupt; Hershey lost $466M in revenue during Halloween from a bad ERP rollout)
• Make smarter decisions about mergers and acquisitions — McKinsey found many M&As fail because firms can't merge incompatible IT systems

A firm without technologically literate leaders in the boardroom is setting itself up for failure.
The Software Layer Cake
Think of software as a layer cake — each layer depends on the one below it:
👤 Users
The people using the system
📊 Applications (Desktop & Enterprise)
Word, Excel, ERP, CRM, BI — where real work happens
⚙️ Operating System
Windows, macOS, iOS, Android — controls hardware, sets standards
🔧 Hardware
CPU, RAM, storage, networking — the physical foundation
The OS creates a stable platform so applications don't have to worry about managing hardware directly. Applications build on top of the OS's services. Users interact only with the top layer — but decisions made at every layer affect cost, security, flexibility, and performance.
The Four Ways to Classify Software
When someone asks "what kind of software is it?", there are four useful lenses:

1. By role in the stack: Is it an OS, a middleware/database layer, or an application?
2. By user scope: Is it desktop software (one user) or enterprise software (many users across an org)?
3. By deployment: Is it installed locally, stored on firmware/chips, or delivered via the cloud (SaaS)?
4. By ownership model: Is it proprietary (paid, closed-source), open source (free, community-built), or custom-built in-house?

Each classification has different implications for cost, vendor lock-in, security, and strategic flexibility.
What the OS Does — For Users and Programmers
For users: The OS creates a consistent, predictable visual environment. Scroll bars, save dialogs, and file browsers behave the same across all apps — reducing learning time, training costs, and errors. You don't need to learn a new interface for every program.

For programmers: The OS provides a common set of commands to interact with hardware. A game developer building for Nintendo Switch doesn't write their own code to play sounds or save files — they call the OS's built-in tools. This makes software faster to write, more reliable, and cheaper to develop. When Apple gave developers strong OS tools via iOS and a distribution channel via the App Store, 50,000 apps appeared in under a year — a textbook example of how a well-designed OS creates platform network effects.
ERP: The Business Functions It Touches
A well-implemented ERP system can coordinate nearly every department in a company simultaneously:
💰 Sales
Verify credit, calculate pricing in any currency, generate orders automatically
📦 Inventory
Check real-time stock levels across warehouses during a sales call
🏭 Manufacturing
Trigger production orders automatically when a sale is confirmed
👥 Human Resources
Flag hiring needs when order volume exceeds current workforce capacity
🛒 Purchasing
Auto-reorder raw materials from suppliers when inventory runs low
📊 Decision Support
Surface trends (e.g., European sales are booming) to guide strategy
The Real Risks of ERP: Don't Ignore the People Layer
ERP installations are notoriously difficult. The average large company spends ~$15 million, and failures are common and catastrophic:

• FoxMeyer Drug — $6B company. Failed ERP led to bankruptcy; sold for under $80M one year later.
• Hershey Foods — Botched rollout caused $466M revenue shortfall; couldn't get candy to stores before Halloween.
• Avon — Spent $125M on a system sales reps refused to use; mass rep exodus followed.

The pattern in every failure: firms focused obsessively on the software and hardware but ignored the equally critical components — people, data, and procedures. IS is made of five components: hardware, software, data, people, and procedures. Lose sight of any one and the whole system can collapse.
Embedded Systems & Otis Elevator — Strategy in a Chip
Otis Elevator embeds software in its elevators, escalators, and walkways that automatically contacts Otis service centers when maintenance is needed. This creates multiple layers of competitive advantage:

• Switching costs: Embedded systems make it harder for third parties to service Otis equipment, locking in the service revenue stream
• Cost efficiency: Maintenance is triggered by actual need (low lubricant, worn parts), not guessed schedules
• Customer satisfaction: Technicians are dispatched before clients even know there's a problem
• Product improvement: Failure data feeds back to R&D to build better future products

This is Moore's Law creating strategic value — cheap chips embedded in physical products, generating information advantages that competitors struggle to replicate.

C

D
The OS only benefits end users through the GUI — it provides no advantage to programmers.
✓ Correct! This is exactly what an OS does for programmers. By providing standardized commands to control hardware (audio, storage, display), the OS lets developers focus on their actual product rather than re-solving solved problems. It also means games are more consistent, reliable, and easier to test.
✗ Not quite. One of the OS's two big jobs is providing programmers with a standard set of hardware-control tools. Without this, every developer would have to write their own audio engine, file system handler, etc. — massively increasing cost, time, and the chance of errors.

Question 4
Otis Elevator's embedded systems automatically alert service centers when an elevator needs maintenance — often before the client notices any problem. Beyond just convenience, which strategic advantage does this MOST directly create for Otis?
A
It reduces Otis's manufacturing costs by allowing cheaper parts to be used.
B
It creates switching costs by making third-party servicing more difficult, locking in a recurring service revenue stream that competitors cannot easily replicate.
C
It allows Otis to charge more for elevators at the point of sale.
D
It primarily benefits customers with no direct financial benefit to Otis.
✓ Correct! Embedded systems create switching costs — because the system is proprietary, third-party repair services can't easily compete with Otis's own technicians who receive the automated alerts. This locks in a lucrative service business on top of the hardware sale, and it gets better over time as R&D uses the failure data to build more reliable products.
✗ Not quite. The big strategic win here is switching costs and service lock-in. The embedded system creates a closed loop: only Otis receives the alerts, so only Otis (or authorized partners) can respond efficiently. This makes it structurally harder for third-party service firms to compete — which is worth far more strategically than just a customer convenience feature.
